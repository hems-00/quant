# Hybrid Market Risk Pipeline

> **LightGBM + FinBERT** — Classifies Nifty 50 daily market risk into Low / Medium / High using technical market features and NLP-derived news sentiment signals. Includes SHAP-based explainability, geopolitical sentiment coverage, and a persistent risk history log.

## What This Does

Each day, the pipeline:

1. Loads Nifty 50 OHLCV data and computes 3 technical features
2. Fetches the latest market + geopolitical news headlines via NewsAPI
3. Scores each headline using FinBERT and aggregates them into 5 daily sentiment signals
4. Trains (or loads cached) a LightGBM classifier on the combined 8-feature dataset
5. Predicts today's risk class: **Low / Medium / High**
6. Explains the prediction using SHAP feature attribution
7. Saves a dashboard, a SHAP plot, a full report, and a persistent risk history CSV

The risk label is derived from 21-day realised volatility tertiles — the bottom third of days are Low risk, middle third are Medium, top third are High.

## Setup

### 1. Clone and create environment

```bash
git clone https://github.com/your-username/hybrid-market-risk-pipeline.git
cd hybrid-market-risk-pipeline

python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

**Core dependencies:**

| Package | Purpose |
|---|---|
| `lightgbm` | Risk classifier |
| `transformers`, `torch` | FinBERT sentiment model |
| `shap` | Feature attribution |
| `pandas`, `numpy` | Data manipulation |
| `yfinance` | Nifty 50 data download |
| `requests` | NewsAPI calls |
| `joblib` | Model caching |
| `python-dotenv` | Environment variable loading |
| `matplotlib` | Dashboard plots |

### 3. Create your `.env` file

```bash
# .env — never commit this file
NEWS_API_KEY=your_newsapi_key_here
```

Get a free NewsAPI key at [https://newsapi.org](https://newsapi.org).  
The free tier allows 100 requests/day and 30 days of history.

### 4. (Optional) Download FinBERT locally

The sentiment module will auto-download `yiyanghkust/finbert-tone` from HuggingFace on first run (~450MB). To cache it locally for offline use:

```python
from transformers import AutoTokenizer, AutoModelForSequenceClassification
tokenizer = AutoTokenizer.from_pretrained("yiyanghkust/finbert-tone")
model = AutoModelForSequenceClassification.from_pretrained("yiyanghkust/finbert-tone")
tokenizer.save_pretrained("C:/hf_models/finbert")
model.save_pretrained("C:/hf_models/finbert")
```

## How To Run


### Step 1 — Generate Market Features

```bash
python src/data_loader.py
```

**What it does:**
- Downloads Nifty 50 OHLCV data via yfinance (2016 to present)
- Computes EMA crossover signal, volume Z-score, and normalised ATR
- Saves to `data/processed/nifty_features.csv`

**Expected output:**
```
[DataLoader]  Downloaded 2443 rows of Nifty 50 data
[DataLoader]  Features computed: ema_signal, volume_z, atr_norm
[DataLoader]  Saved to data/processed/nifty_features.csv
```

**Only needs to be re-run when:** you want to add more recent market data (run daily or weekly).

---

### Step 2 — Generate Sentiment Signals

```bash
python src/sentiment.py
```

**What it does:**
- Fetches the last 7 days of headlines from NewsAPI across 19 queries covering:
  - Indian market indices (Nifty, Sensex, Bank Nifty)
  - Macro signals (crude oil, RBI, USD/INR, inflation)
  - Geopolitical events (Russia-Ukraine, Middle East, India-China)
- Scores each headline using FinBERT (positive − negative probability)
- Applies 1.5× weight to geopolitical headlines to correct for FinBERT's financial-domain bias
- Aggregates into 5 daily signals and saves to `data/processed/latest_news_sentiment.csv`

**Expected output:**
```
[Fetch]  Using 19 queries (6 market + 7 macro + 6 geopolitical)
[Fetch]  Fetching news from 2026-02-22 (7 days)...
FinBERT loaded on cuda
Scoring 187 headlines...
  Applied 1.5x geopolitical weight to 43 headlines.

[Sentiment] Signal ranges for this batch:
  news_impact            min=-0.8431  max=+0.6219  mean=-0.1203
  news_uncertainty       min=+0.0000  max=+5.3224  mean=+1.8841
  negative_shock         min=+0.0000  max=+2.1580  mean=+0.4432
  sent_momentum          min=-0.4891  max=+0.3012  mean=-0.1067
  headline_intensity     min=+0.2764  max=+2.2553  mean=+1.3210

Saved to data/processed/latest_news_sentiment.csv
```

**Only needs to be re-run when:** you want updated news sentiment (run daily).

> **Note on sentiment coverage:** The pipeline trains on all available market history (~2443 days) but sentiment data only covers the days you have run this script. The more historical runs you accumulate in `latest_news_sentiment.csv`, the more the model learns from sentiment signals. On first run, the model trains effectively on market features only and ignores sentiment features — this is expected and correct behaviour.

---

### Step 3 — Run The Pipeline

```bash
# Standard run (uses sentiment if available)
python run_pipeline.py

# Market features only (skip sentiment)
python run_pipeline.py --no-sentiment

# Force model retraining (use after adding new data)
python run_pipeline.py --retrain

# Run COVID-2020 crisis backtest after prediction
python run_pipeline.py --crisis-backtest

# Force retrain + crisis backtest combined
python run_pipeline.py --retrain --crisis-backtest
```

## Feature Schema

### Market Features (3)

| Feature | Description | Source |
|---|---|---|
| `ema_signal` | EMA crossover signal — trend direction indicator | `src/data_loader.py` |
| `volume_z` | Volume Z-score vs rolling 21-day baseline | `src/data_loader.py` |
| `atr_norm` | Average True Range normalised by price — volatility measure | `src/data_loader.py` |

### Sentiment Features (5)

| Feature | Description | Formula |
|---|---|---|
| `news_impact` | Signed net sentiment weighted by article volume | `sent_mean × log1p(count)` |
| `news_uncertainty` | Cross-headline disagreement — market confusion proxy | `sent_std × sqrt(count)` |
| `negative_shock` | Adverse event magnitude — fires on crisis headlines | `clip(-sent_mean, 0) × log1p(count)` |
| `sent_momentum` | Directional persistence of sentiment trend | `EWM(sent_mean, span=3)` |
| `headline_intensity` | News volume relative to 30-day rolling baseline | `count / rolling_30d_mean` |

### Risk Label

Derived from 21-day realised volatility (`rv_21`) tertiles computed on the full training window:

| Class | Label | Definition |
|---|---|---|
| 0 | Low Risk | `rv_21` below 33rd percentile |
| 1 | Medium Risk | `rv_21` between 33rd and 66th percentile |
| 2 | High Risk | `rv_21` above 66th percentile |

## Model Details

**Algorithm:** LightGBM multiclass classifier (`objective=multiclass, num_class=3`)

**Validation:** TimeSeriesSplit with 5 folds — each fold trains only on past data and validates on future data, preserving temporal order. Standard K-Fold is not used because random shuffling would allow the model to train on future data and validate on the past, producing misleading accuracy estimates.

**Key hyperparameters:**

| Parameter | Value | Reason |
|---|---|---|
| `n_estimators` | 500 | Enough trees for complex patterns at this data size |
| `learning_rate` | 0.05 | Conservative — reduces overfitting at 500 trees |
| `max_depth` | 5 | Constrained tree depth prevents overfitting |
| `num_leaves` | 31 | Leaf-wise cap: 31 < 2^5 ensures bounded complexity |
| `min_child_samples` | 20 | No leaf with fewer than 20 samples — prevents noise memorisation |
| `class_weight` | balanced | Auto-adjusts for any class imbalance in the label distribution |

**Caching:** The trained model is saved to `models/lgbm_risk_model_full.pkl`. On subsequent runs it is loaded directly, skipping the training step. If the feature count changes (e.g. sentiment data added or removed), the pipeline detects the mismatch via `n_features_in_`, deletes the stale cache automatically, and retrains.

## Known Limitations

**Sentiment data coverage:** The model achieves meaningful sentiment signal only after accumulating enough historical sentiment rows. On first run, all 5 sentiment SHAP values will be zero — this is expected. The model correctly ignores features that are near-constant across the training window. As you run `src/sentiment.py` daily and append to `latest_news_sentiment.csv`, the model progressively learns from sentiment features. Full signal typically emerges after ~100 days of data.

**NewsAPI free tier:** Limited to 100 requests/day and articles from the past 30 days. With 19 queries at 50 articles each, a single `sentiment.py` run uses 19 requests. This leaves comfortable headroom for daily use. The paid Developer plan ($449/month) extends history to 24 months, which would significantly improve sentiment coverage.

**FinBERT domain bias:** FinBERT was trained on financial filings and earnings calls. It scores war and geopolitical headlines less accurately than financial market language. The 1.5× geopolitical weight in `process_headlines()` partially corrects for this but is not a substitute for a fine-tuned model. Validation against your ground-truth market outcomes is recommended after accumulating 30+ days of data.

**Look-ahead guard:** All features are shifted by 1 day before training (`shift(1)` in `merge_features()`). The model always predicts today's risk using yesterday's features. This means the pipeline cannot produce same-day sentiment predictions — it reflects confirmed signals from the previous close.
